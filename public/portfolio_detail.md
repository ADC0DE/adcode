## 2. 질문별 처리 방식
| 항목 | 처리 |
|---|---|
| **기존 9개 + 신규** | 받은 33개가 **기존 9개를 모두 포함**. 중복 없이 **33개로 통합**하되, **기존 9개를 앞(1~9번)으로 정렬**하고 신규 24개를 10~33번에 배치. 원고는 클라이언트 정식본으로 교체 |
| **연도·클라이언트** | 둘 다 있으면 모두 표기. **공란 7개는 추론해 채우고 "(추정)" 표기는 제거**(아래 3번) |
| **단일 이미지 크기·정렬** | 작업이미지는 **가운데 정렬**. SNS 단일 이미지(예: 추석 과일선물세트)는 좁은 격자 대신 **가운데 큰 단독 배치**(`pf-images--solo`, 최대폭 900px)로 키움 |
| **클라이언트 2곳(·)** | "우편나라 · 판다서점"처럼 **그대로 표기** |
| **목록 순서** | **받은 순서대로**(SNS→광고소재→상세페이지) |
| **SNS 이미지 비율** | **원본 비율 유지**(잘림 없음) — 메이슨리 컬럼 배치 |
| **브랜딩·콘텐츠 필터** | 작업물 0개지만 **탭 그대로 유지**(클릭 시 "준비 중") |
| **더보기** | 처음 **9개 노출 → 클릭 시 9개씩** 추가 (33개라 정상 작동) |
| **제목** | txt 제목 **그대로 사용**(예: "사과 상세페이지", "보험 롤링배너") |
| **이미지 최적화** | 원본 **193MB → 48MB**로 리사이즈+고압축 (썸네일 800px / 작업이미지 가로 최대 1080px). webp는 변환기 부재 + 일부 상세이미지가 webp 최대높이(16383px) 초과로 부적합 → **JPEG 고압축** 채택 |
